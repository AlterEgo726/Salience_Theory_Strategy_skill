# ST因子（Salience Theory Value）策略回测工具

## 📖 什么是ST因子？

**ST因子（Salience Theory Value，凸显理论价值）** 是基于行为金融学中**凸显理论（Salience Theory）** 构建的因子。

### 核心思想

投资者在决策时，会对**凸显收益**（即明显偏离市场平均水平的收益）赋予**过高的心理权重**，而对普通收益赋予较低权重。

### 计算公式

对于个股 $i$ 在交易日 $t$ 的日收益率 $R_{i,t}$ 和市场日收益率 $R_{m,t}$：

1. **凸显度（Salience）**：$\displaystyle \sigma_{i,t} = \frac{|R_{i,t}-R_{m,t}|}{|R_{i,t}|+|R_{m,t}|+\theta}$

2. **排序**：按月对每只股票的各交易日按凸显度降序排列

3. **决策权重**：$\displaystyle w_{i,k} = \frac{\delta^{k}}{\sum_j \delta^{j}}$，其中 $k$ 为排序序号，$\delta$ 为凸显权重参数

4. **ST值**：$\displaystyle ST_i = \sum_t(p_t \cdot w_{i,t} \cdot R_{i,t}) - \sum_t(p_t \cdot R_{i,t})$

其中 $p_t = 1/T$ 为等权时间权重，$\theta$ 和 $\delta$ 为可调参数（默认 $\theta=0.1$, $\delta=0.7$）。

> **直观理解**：ST因子衡量的是"凸显加权收益"与"简单平均收益"之差。如果某只股票在凸显交易日获得了较高的正收益，其ST值就为正；反之，如果在凸显交易日亏损严重，ST值为负。

## 📊 策略逻辑

每月根据个股的**ST值**进行排序，分为10组：

| 分组 | ST值 | 策略角色 |
|:---:|:----:|:--------:|
| **组1** | **最低**（ST 最负） | **做多** 🟢 — 凸显亏损，可能被低估 |
| **组10** | **最高**（ST 最正） | **做空** 🔴 — 凸显盈利，可能被高估 |
| **组2~9** | 中间 | 填充组 |

**多空组合**：做多组1 + 做空组10 = **Low-High (L-H) 组合**

## 🗂️ 数据需求

### 基础运行（等权/马科维茨共用）

| 文件 | 格式 | 必需列 | 说明 |
|:----|:----|:------|:----|
| `stock_ret_df.csv` ✅ | CSV | `Stkcd`, `date`, `Ri` | **个股日收益率**（非百分数，如0.01=1%） |
| `market_ret_df.csv` ✅ | CSV | `date`, `Rm` | **市场等权日收益率**（非百分数） |

有这两个文件即可完成**ST因子计算 + 等权回测**。

### 马科维茨组合额外需求

如果选择 **马科维茨最小方差组合** 策略，需要 `stock_ret_df.csv` 覆盖的时间范围满足：

```
起始日期 ≤ 回测开始日期 - 36个月（默认回溯窗口）
```

因为马科维茨需要回溯36个月的日度收益率数据来估计协方差矩阵。例如，如果回测从2015年1月开始，则日度数据至少要从2012年1月开始。

> ⚠️ **运行时间警告**：马科维茨策略需要对**每个月份×每个分组**（约数百个组合）逐一进行协方差矩阵估计→对角收缩→SLSQP优化求解权重，**运行时间可能很长**（对于10年数据，可能需数十分钟甚至更久）。建议：
> - 先用**等权策略**快速验证
> - 确认数据无误后再跑马科维茨
> - 如果只关心L-H组合，可仅跑组1和组10

### ⏱️ 最少数据时间要求

| 策略 | 最少需要 | 说明 |
|:----|:--------|:----|
| **等权（无/有成本）** | **2个月的日度数据** | 第1个月算ST因子，第2个月作为下月收益（`ret_next`） |
| **马科维茨（无/有成本）** | **起始日再早36个月** | 在等权的基础上，需要额外回看36个月日度数据估算协方差矩阵 |

**示例**：
- 要回测 **2025年1月~2025年12月** 的等权策略 → 需 `stock_ret_df.csv` 覆盖 **2025-01~2025-12**（12个月）
- 要回测同样时段的马科维茨策略 → 需 `stock_ret_df.csv` 覆盖 **2022-01~2025-12**（48个月）

> **提示**：如果数据长度有限，可在 `config.json` 中调小 `lookback_months` 来降低马科维茨的窗口要求。

**等权策略（基准需求）**

| 阶段 | 需要的时间 |
|:----|:----------:|
| 计算ST因子 | ≥ 1个月的日度数据（按月分组） |
| 月度收益 + ret_next | 需要多1个月的收益作为下月收益 |
| **合计最少** | **2个月的日度数据** |

**马科维茨策略（额外需求）**

等权的2个月基础上，还要额外回看36个月（默认）的日度数据来算协方差。默认情况下数据需覆盖 **38个月以上**。可以通过 `config.json` 调小 `lookback_months` 和 `min_history_days`：

```json
{
    "lookback_months": 12,
    "min_history_days": 168
}
```

### 数据文件存放位置

将数据文件放入 `data/input/` 目录下。

## 🎯 输出结果

### 主输出：`pivot.csv`

一个以 `month` 为索引的DataFrame，每行代表一个月度，各列含义：

| 列名 | 含义 |
|:----|:----|
| `month` | 月份（时间戳格式） |
| `1` | **第1组（ST最低组）** 月度收益 — 做多目标 |
| `2`~`9` | 第2~9组月度收益 |
| `10` | **第10组（ST最高组）** 月度收益 — 做空目标 |
| `L-H` | **多空组合收益** = 组1收益 − 组10收益（已扣除交易成本，若启用） |
| `H-L` | **反向多空收益** = 组10收益 − 组1收益（已扣除交易成本，若启用） |

### 附加输出

- 各组及多空组合的**平均收益**（打印在控制台）
- 若启用成本，所有收益已**扣除双边交易成本**

## ⚙️ 四种策略组合

| 编号 | 组合权重 | 考虑成本 | 启动方式 |
|:---:|:--------:|:--------:|:--------|
| ① | **等权** | ❌ 无成本 | `python main.py -s equal_weight -c False` |
| ② | **等权** | ✅ 有成本 | `python main.py -s equal_weight -c True` |
| ③ | **马科维茨** | ❌ 无成本 | `python main.py -s markowitz -c False` |
| ④ | **马科维茨** | ✅ 有成本 | `python main.py -s markowitz -c True` |

### 等权组合
每月对组内所有股票取等权平均收益，简单快速。

### 马科维茨最小方差组合
通过历史收益率估计协方差矩阵 → 对角收缩（shrink=0.2）→ 最小方差优化（long-only，单股权重上限10%），得到组内最优权重。

### 交易成本
单边交易成本通过 `config.json` 中的 `cost_rate` 参数控制（默认 0.002 = 0.20%）。每月调仓时根据换手率计算双边成本：

```
月度成本 = cost_rate × (买入换手率 + 卖出换手率)
```

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 准备数据

将 `stock_ret_df.csv` 和 `market_ret_df.csv` 放入 `data/input/` 目录。

### 3. 运行回测

```bash
# ① 等权无成本（最快）
python main.py -s equal_weight -c False

# ② 等权有成本
python main.py -s equal_weight -c True

# ③ 马科维茨无成本（运行时间长！）
python main.py -s markowitz -c False

# ④ 马科维茨有成本（运行时间长！）
python main.py -s markowitz -c True
```

### 4. 查看结果

- 输出CSV在 `output/pivot.csv`
- 控制台打印各组平均收益

## 🔧 配置参数

通过 `config.json` 调整：

| 参数 | 默认值 | 说明 |
|:----|:------|:----|
| `theta` | 0.1 | 凸显函数平滑参数 |
| `delta` | 0.7 | 凸显权重衰减参数 |
| `lookback_months` | 36 | 马科维茨回溯窗口（月） |
| `min_history_days` | 504 | 股票最少历史交易天数 |
| `max_weight` | 0.10 | 马科维茨单股权重上限 |
| `shrink` | 0.20 | 协方差对角收缩系数 |
| `cost_rate` | 0.002 | 单边交易成本（0.20%） |
| `decile_count` | 10 | 每月分组数量 |
